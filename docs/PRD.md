# ConsoleMVC PRD

## 개요
반도체 시료 관리 프로그램의 콘솔 UI 골격. Model / Controller / View 구조로 아래 6개 메인 메뉴를 구성한다.

## 요구사항
- 메인 메뉴: 시료관리 / 시료주문 / 시료승인·거절 / 모니터링 / 출고처리 / 생산라인
- 각 메인 메뉴는 서브 메뉴(시료등록·조회·검색, 시료예약, 접수목록·승인·거절, 주문량·재고량확인, 출고실행, 생산현황·대기주문확인 등)로 진입
- Model : Sample(시료ID, 이름, 평균생산시간, 수율, 재고수량), Order(주문ID, 시료ID, 고객명, 주문수량, 상태) 등 도메인 객체
- Controller : 사용자 입력을 받아 Model/데이터 처리 계층을 호출하고 결과를 View에 전달
- View : 콘솔 입출력만 담당 (메뉴 표시, 목록/상태 표시, 입력값 검증은 Controller 책임)
- 실제 데이터 저장/조회는 DataPersistence, 생산 데이터 조회는 DataMonitor 쪽 구현에 위임 (연동 인터페이스만 이 폴더에서 정의)
